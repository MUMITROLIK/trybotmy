# ML Model Improvement Plan

## Текущая ситуация (ПЛОХО):
- Винрейт в данных: 45.5%
- CV AUC: 0.54 (почти случайность)
- Модель обучена на 696 строках
- Модель переобучена (train AUC 0.88, CV AUC 0.54)

## Проблема:
**Модель учится на мусоре!** Если в данных 45% винрейт, модель не может предсказать лучше.

## Решение: 3-этапный подход

### ЭТАП 1: Очистка данных (КРИТИЧНО!)

Нужно обучать модель ТОЛЬКО на качественных сигналах:

```python
# Фильтры для обучающих данных:
1. Только сигналы с strength >= 65 (убираем слабые)
2. Только сигналы с heavy_confirmed >= 1 (убираем без подтверждения)
3. Только сигналы где BTC тренд совпадает с направлением
4. Убираем азиатскую сессию (низкая ликвидность)
5. Убираем TIMEOUT (не достигли ни TP ни SL)
```

Ожидаемый результат: винрейт в данных вырастет до 55-60%

### ЭТАП 2: Feature Engineering (добавить умные фичи)

Текущие фичи - это "сырые" индикаторы. Нужны УМНЫЕ фичи:

```python
# Новые фичи:
1. trend_alignment = (direction == LONG and btc_trend == bull) ? 1 : 0
2. confluence_score = count(indicators > 0.5)  # сколько индикаторов согласны
3. risk_reward_ratio = (tp1_pct / sl_pct)
4. market_momentum = (volume_ratio * cvd_norm)
5. volatility_regime = (atr_pct > atr_percentile_80) ? "high" : "normal"
6. session_quality = session винрейт из истории
7. symbol_performance = винрейт этого символа за последние 7 дней
```

### ЭТАП 3: Ансамбль моделей (вместо одной)

Вместо одной модели - используем 3:

```python
Model 1: Предсказывает TP1/TP2 (фокус на винах)
Model 2: Предсказывает SL (фокус на лоссах)
Model 3: Предсказывает длительность сделки

Финальное решение = weighted vote
```

## Что делать СЕЙЧАС:

### Вариант A: Быстрое улучшение (1 час)

Создать `ml_train_v2.py` с фильтрами данных:

```python
# Обучаем только на качественных сигналах
df_filtered = df[
    (df['strength'] >= 65) &
    (df['heavy_confirmed'] >= 1) &
    (df['target'].isin(['TP1', 'TP2', 'SL', 'SL_AFTER_TP1'])) &
    (df['session'] != 'asian')
]

# Добавляем trend_alignment
df_filtered['trend_alignment'] = (
    ((df_filtered['direction'] == 'LONG') & (df_filtered['btc_trend_4h'] == 'bull')) |
    ((df_filtered['direction'] == 'SHORT') & (df_filtered['btc_trend_4h'] == 'bear'))
).astype(int)
```

Ожидаемый результат: CV AUC вырастет до 0.60-0.65

### Вариант B: Правильное решение (2-3 часа)

1. Собрать БОЛЬШЕ данных (1500+ строк)
2. Применить фильтры из Варианта A

5. Добавить калибровку вероятностей

Ожидаемый результат: CV AUC 0.65-0.70, реальный винрейт 55%+

### Вариант C: Временное решение (10 минут)

Пока модель плохая - **отключи ML фильтр** и используй только rule-based фильтры:

```python
# В .env:
ML_FILTER_ENABLED=false

# Используй вместо ML:
MIN_SIGNAL_STRENGTH=70
SIGNAL_CATEGORY_FILTER=['PREMIUM', 'STANDARD']
BLOCK_LONG_ON_BEAR=true
BLOCK_SHORT_ON_BULL=true
```

Это даст лучший результат чем плохая ML модель!
