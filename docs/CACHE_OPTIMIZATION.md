# Оптимизация скорости сканирования — WebSocket кэш свечей

## Проблема

**До оптимизации:**
- Каждый скан делал 50+ REST запросов к Binance
- Время скана: ~75 секунд
- Binance rate limit: 1200 req/min (легко превысить)

## Решение

**WebSocket кэш свечей** — держит актуальные данные в памяти, обновляется в реальном времени.

## Результаты теста

```
Инициализация: 5.47s (один раз при старте)
Чтение из кэша: 0.5ms на монету
REST запрос: 310ms на монету

Ускорение: 619x 🚀
```

## Архитектура

```
Startup:
  ├─ REST загрузка 50 монет параллельно (15 сек, один раз)
  └─ WebSocket держит соединение и обновляет кэш в фоне

Каждый час:
  └─ Автообновление списка монет (новые листинги попадают в кэш)

Каждый скан:
  ├─ Читаем свечи из кэша (мгновенно)
  ├─ Остальные данные через REST (orderbook, OI, funding)
  └─ Анализ сигналов

Итого: ~40 секунд вместо 75 секунд
```

## Автообновление списка монет

Каждый час кэш проверяет топ-50 монет и:
- Добавляет новые листинги (загружает свечи)
- Удаляет монеты, вышедшие из топа
- Логирует изменения

**Преимущества:**
- Новые горячие монеты попадают в кэш автоматически
- Не нужно перезапускать бота
- Минимальная нагрузка (1 раз в час)

## Файлы

- `core/candle_cache.py` — кэш свечей
- `main.py` — интеграция в сканер
- `test_cache.py` — тест производительности

## Использование

```python
# Инициализация (в main.py)
from core.candle_cache import init_candle_cache
symbols = await get_top_futures_async(50)
cache = await asyncio.to_thread(init_candle_cache, symbols)

# Чтение (в scan_one)
from core.candle_cache import get_cached_candles
df = get_cached_candles("BTCUSDT")  # мгновенно
```
